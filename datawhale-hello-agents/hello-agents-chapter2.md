## 2. 第二章主题和学习目标

第二章主题是智能体发展史。它不是直接讲现代 LLM Agent，而是先回到人工智能早期，理解现代智能体的设计思想从哪里来。

本章核心线索：

```text
符号主义与逻辑规则
-> 专家系统与微观世界智能体
-> 符号主义的局限
-> 分布式与协作式智能
-> 学习范式
-> LLM 驱动的现代智能体
```

关键理解：

> 智能体的发展史是问题驱动的。每一代范式都是为了解决上一代范式的痛点，但新方案又会带来新的局限，于是推动下一代范式出现。

今天纠正的一个理解：

> LLM 不是天生等于 Agent。LLM 更像现代 Agent 的“大脑候选”。当它接上记忆、规划、工具、环境反馈之后，才成为 LLM 驱动的 Agent。

## 3. 符号主义：基于规则的智能

符号主义认为，人类智能，尤其是逻辑推理能力，可以被形式化的符号体系捕捉和复现。

它的基本想法是：

```text
把世界表示成符号
-> 把知识写成规则
-> 用推理规则推出新结论
-> 根据结论规划行动
```

早期符号主义智能体的“聪明”，主要来自：

- 人类专家手工编码的知识库；
- 人类设计好的推理规则；
- 系统对这些符号和规则的计算。

一个简单例子：

```text
IF 病人发烧 AND 咳嗽 THEN 可能患有呼吸道感染
```

这里的系统不是从大量病例里自己学出规律，而是根据人提前写好的规则做推理。

## 4. 符号主义与学习范式的区别

今天对比了符号主义和学习范式：

| 范式 | 能力来源 | 典型方式 | 优势 | 局限 |
| --- | --- | --- | --- | --- |
| 符号主义 | 人工编码的知识和规则 | `IF-THEN`、逻辑推理、知识库 | 可解释、可控、适合边界清楚的领域 | 规则写不完，遇到开放世界和例外情况脆弱 |
| 学习范式 | 数据、经验和学习算法 | 神经网络、深度学习、强化学习 | 能从数据中学习模式，适合图像、语音、自然语言等复杂输入 | 很多系统偏专用，缺少通用语言接口和广泛世界知识 |

关键理解：

> 符号主义的能力来自“人把知识写进去”；学习范式的能力来自“系统从数据和反馈里学出来”。

## 5. 物理符号系统假说

物理符号系统假说，英文是 Physical Symbol System Hypothesis, PSSH，由 Allen Newell 和 Herbert A. Simon 在 1976 年提出。

它的核心主张：

> 智能的本质，就是符号的计算与处理。

两个核心论断：

| 论断 | 含义 |
| --- | --- |
| 充分性论断 | 任何一个物理符号系统，都具备产生通用智能行为的充分手段。也就是：只要系统能操作符号，原则上就可能产生智能。 |
| 必要性论断 | 任何一个能展现通用智能行为的系统，本质上必然是一个物理符号系统。也就是：如果一个东西真的有智能，底层一定在做某种符号操作。 |

PSSH 的意义：

> 它把“智能是什么”这个模糊的哲学问题，转化成了知识表示、符号操作和推理算法这些可以工程化的问题。

## 6. 专家系统

专家系统是符号主义时代最重要、最成功的应用成果之一。

它的核心目标：

> 把某个领域里人类专家的知识和经验编码进计算机，让系统在这个领域里像专家一样给建议或结论。

专家系统的两个核心组件：

| 组件 | 作用 |
| --- | --- |
| 知识库 Knowledge Base | 存放领域专家的知识和经验，常见形式是 `IF-THEN` 产生式规则 |
| 推理机 Inference Engine | 根据用户提供的事实，在知识库中寻找并应用相关规则，从而推出新结论 |

专家系统体现了符号主义 AI 的典型设计思想：

> 知识和推理相分离。知识库负责“知道什么”，推理机负责“怎么推”。

## 7. 正向链与反向链

专家系统的推理机主要有两种工作方式。

### 7.1 正向链

正向链 Forward Chaining 是从已知事实出发，一步步往前推。

例子：

```text
已知事实：发烧、咳嗽
规则：IF 发烧 AND 咳嗽 THEN 可能呼吸道感染
推出新事实：可能呼吸道感染
```

正向链是数据驱动的推理方式，因为它从已有数据和事实开始，向可能结论推进。

### 7.2 反向链

反向链 Backward Chaining 是从目标或假设出发，倒着问：要证明这个目标，需要满足哪些条件？

例子：

```text
目标：病人是否患有肺炎？
倒推：需要发烧、咳嗽、胸片异常、白细胞升高等证据
```

反向链是目标驱动的推理方式，因为它围绕一个待证明的结论寻找证据。

最简单的区别：

```text
正向链：我已经知道这些事实，所以我能推出什么？
反向链：我要证明这个结论，所以我还需要知道什么？
```

## 8. MYCIN 系统

MYCIN 是斯坦福大学在 20 世纪 70 年代开发的专家系统，用来辅助医生诊断细菌性血液感染，并推荐合适的抗生素治疗方案。

它的关键特点：

- 通过问答式交互收集病人的症状、病史和化验结果；
- 知识库包含约 600 条医学专家提供的 `IF-THEN` 规则；
- 主要采用反向链推理，从“确定致病菌”这个高层目标倒推所需证据；
- 引入置信因子 Certainty Factor, CF，用 -1 到 1 的数值表达结论可信度。

CF 解决的问题：

> 医学诊断不是非黑即白。很多证据只是增强或削弱某个判断，而不是 100% 证明。置信因子让系统能够表达和计算不确定性。

MYCIN 的意义：

> 它证明了精心设计的知识工程和符号推理，可以让机器在边界清楚、规则相对稳定的专业领域中表现出很强的能力。

MYCIN 的限制：

> 它适合垂直领域，但很难直接扩展到开放世界。真实世界存在大量模糊语境、例外情况和难以编码的隐性知识。

## 9. SHRDLU

SHRDLU 是 Terry Winograd 在 1968-1970 年开发的早期符号 AI 项目。它运行在一个叫 Blocks World 的积木世界中。

这个世界里有：

- 不同颜色、形状、大小的积木；
- 一个可以抓取和移动积木的虚拟机械臂；
- 用户可以用自然语言向系统下命令或提问。

如果说 MYCIN 展示的是符号 AI 在专业领域里的深度，那么 SHRDLU 展示的是：

> 在一个简化世界中，把自然语言理解、规划、记忆和行动执行集成到同一个系统里的综合能力。

SHRDLU 像现代 Agent 早期雏形的原因：

```text
用户给出自然语言指令
-> 系统理解目标
-> 判断当前环境状态
-> 规划动作路径
-> 执行动作
-> 记录行为和上下文
-> 回答后续问题
```

这已经非常接近现代 Agent 的基本轮廓：

```text
目标 -> 理解 -> 规划 -> 行动 -> 观察/记忆 -> 继续调整
```

但 SHRDLU 的局限也很明显：

> 它的成功高度依赖一个小而封闭、规则清楚、对象有限的积木世界。真实世界远比积木世界复杂，无法完全靠人手写符号和规则覆盖。

## 10. 今日问答纠偏

今天重点纠正了几个理解：

- 不要把符号主义的知识库理解成“喂大量训练数据”。它更准确是人工编码的知识和规则。
- “充分性论断 / 必要性论断”不是“垄断”。
- 学习范式虽然能从数据中学习，但很多传统学习系统偏专用，不适合作为通用 Agent 大脑。
- LLM 主要提供通用语言理解、知识压缩、上下文推理和任务泛化能力；工具使用是把 LLM 放进 Agent 架构后，通过工具接口和执行循环接上的。
- MYCIN 诊断的是细菌性血液感染，不是泌尿系统感染。
- CF 是置信因子，用于处理不确定性，不只是改变用户预期。

## 12. 2.1.4 符号主义面临的根本性挑战

符号主义在专家系统和 SHRDLU 这类封闭环境里取得过成功，但当它从“微观世界”走向开放、复杂的真实世界时，会遇到根本难题。

原文把这些挑战分成两大类：

1. 常识知识与知识获取瓶颈
2. 框架问题与系统脆弱性

### 12.1 知识获取瓶颈

知识获取瓶颈不是简单地说“知识太多”，而是指把知识放进系统的过程很难：

- 需要专家和知识工程师访谈、提炼、编码；
- 成本高、耗时长、难以规模化；
- 很多专家知识是隐性的、直觉性的，很难写成清楚的 `IF-THEN` 规则。

例子：

```text
一个老医生说：“这个病人看起来不太对劲。”
```

这句话背后可能包含经验、直觉、视觉观察、病史联想等复杂判断，但很难直接变成：

```text
IF 条件 A AND 条件 B AND 条件 C THEN 结论 X
```

### 12.2 常识问题

常识问题的重点是：真实世界需要海量默认背景知识，而符号系统如果没有被明确编码，就对此一无所知。

比如：

```text
水是湿的。
玻璃容易碎。
绳子可以拉，但不能推。
人进门通常要先开门。
杯子倒过来，里面的水可能会流出来。
```

人类默认知道这些事情，但符号系统需要一条条写进去。

一句话区分：

```text
知识获取瓶颈 = 知识怎么录进去很难。
常识问题 = 要录进去的常识世界太大、太碎，几乎录不完。
```

### 12.3 框架问题

框架问题问的是：

> 在一个动态世界中，智能体执行一个动作后，系统如何高效判断哪些东西变了，哪些东西没变？

比如机械臂移动一个红色积木。

变化的是：

```text
红色积木的位置
机械臂当前状态
桌面部分空间占用情况
```

没变的是：

```text
红色积木还是红色
红色积木形状没变
蓝色积木位置没变
桌子还在
房间灯光没变
```

人类会自动忽略大量无关变化，但符号系统如果没有高效机制，就可能需要为每个动作显式声明大量“不变项”。真实世界对象、属性和动作太多，这在计算上不可行。

### 12.4 系统脆弱性

系统脆弱性指的是：符号系统强依赖预设规则，一旦遇到规则之外的新情况，哪怕变化很小，也可能完全失灵。

今天用车载导航做了类比：

```text
“导航到 XX 地方” -> 系统能理解
“我要去 XX 地方，怎么走？” -> 如果规则没覆盖，系统可能无法触发导航
```

这里的问题不是系统没有导航能力，而是入口表达没有被规则覆盖。

关键理解：

> 符号主义的优势是清晰、可解释、可控；但它很容易被开放世界里的模糊表达、隐性知识、例外情况和动态变化击穿。

## 13. 2.2 构建基于规则的聊天机器人：ELIZA

原文接着用 ELIZA 这个早期聊天机器人，让我们直观感受规则系统如何“假装理解人话”。

### 13.1 ELIZA 的设计思想

ELIZA 是 Joseph Weizenbaum 在 1966 年发布的程序。它最著名的脚本是 `DOCTOR`，模仿非指导性心理治疗师。

ELIZA 的核心机制：

```text
识别关键词
-> 套用预设转换规则
-> 把用户的话改写成开放式问题
```

例子：

```text
用户：我为我的男朋友感到难过
ELIZA：你为什么会为你的男朋友感到难过？
```

ELIZA 并不真正理解用户，而是用句式转换制造“被理解”的体验。

产品经理视角：

> 用户感受到的“理解”，不一定等于系统机制上的真实理解。要区分“体验上像理解”和“真正具备语义理解、目标拆解、行动闭环”。

### 13.2 模式匹配与文本替换

ELIZA 的算法流程可以拆成四步：

1. 关键词识别与排序
2. 分解规则
3. 重组规则
4. 代词转换

压缩理解：

```text
找关键词
-> 按规则拆句子
-> 套模板重组
-> 做代词转换
```

关键词优先级的意义：

> 当一句话同时匹配多个模式时，规则设计者会让更具体、更值得回应的模式优先被处理。

分解规则和重组规则的区别：

```text
分解规则：用模式拆分用户输入，捕获可复用片段。
重组规则：把捕获片段放进回应模板，生成系统输出。
```

代词转换的作用：

```text
I -> you
my -> your
me -> you
```

例如：

```text
I need my mother
```

先匹配并捕获：

```text
my mother
```

代词转换后变成：

```text
your mother
```

最终可以生成：

```text
Why do you need your mother?
```

### 13.3 迷你 ELIZA 的核心逻辑

原文给了一个迷你版 Python 实现。核心结构包括：

- `rules` 规则库；
- `pronoun_swap` 代词转换表；
- `respond(user_input)` 响应函数；
- 主聊天循环。

`rules` 的左边和右边：

```text
左边：匹配模式/正则表达式，用来匹配和捕获用户输入。
右边：回应模板列表，用来生成系统输出。
```

例子：

```python
r'I need (.*)': [
    "Why do you need {0}?",
    "Would it really help you to get {0}?",
    "Are you sure you need {0}?"
]
```

其中 `(.*)` 是捕获组，不是省略号。它会捕获 `I need` 后面的内容，再填进 `{0}`。

`respond(user_input)` 的流程：

```text
遍历 rules
-> 用正则检查 user_input 是否匹配
-> 如果匹配，取出捕获组
-> 对捕获组做代词转换
-> 从回应模板列表里随机选一个
-> 把转换后的捕获内容填进 {0}
-> 返回最终回复
```

### 13.4 规则驱动系统的局限

原文总结了迷你 ELIZA 的三个局限。

第一，缺乏语义理解。

比如：

```text
I am not happy
```

系统只是匹配：

```python
r'I am (.*)'
```

捕获：

```text
not happy
```

它不知道 `not` 改变了语义方向，只是在执行字符串匹配和模板填充。

第二，无上下文记忆。

ELIZA 是无状态的，英文是 `Stateless`。它每次只看当前输入，不记得前面说过什么。

例子：

```text
User: My mother is angry with me.
ELIZA: Tell me more about your mother.
User: She thinks I work too much.
```

人类知道 `She` 指的是 mother，但无状态系统不知道。

第三，规则扩展性问题。

自然语言表达几乎无限：

```text
I am sad.
I feel sad.
I feel kind of down.
I have been emotionally drained.
I am not exactly happy these days.
```

如果每种说法都写规则，规则库会快速膨胀，规则之间还会冲突，优先级管理会越来越复杂。

ELIZA 效应：

> 用户觉得系统懂自己，但本质上是巧妙话术、人类情感投射和规则模板共同制造出的智能幻觉。

产品风险：

> 在心理咨询、医疗、法律、金融等高风险场景里，用户可能把“像理解”误认为“真理解”，从而过度信任系统。

## 14. 2.3 马文·明斯基的心智社会

符号主义和 ELIZA 实践共同暴露出一个问题：单一、集中的推理系统很难走向通用智能。

明斯基在《心智社会》中提出：

> 智能不是来自某个单一、完美、全能的核心，而是来自大量简单部分的多样化协作。

这意味着研究视角从：

```text
一个大脑试图管所有事
```

转向：

```text
很多小模块各管一小块，组合后形成整体智能
```

### 14.1 对单一整体智能模型的反思

20 世纪 70-80 年代，符号主义的局限越来越明显：

- 专家系统在垂直领域有效，但没有儿童般的常识；
- SHRDLU 在积木世界表现很好，但无法理解积木世界之外的事情；
- ELIZA 能模仿对话，但对内容本身一无所知。

这些系统背后都有一种自上而下的思路：

```text
人设计统一规则/中央处理逻辑
-> 系统根据规则处理信息
-> 系统做出决策
```

明斯基开始反思：

```text
理解是不是单一能力？
常识是不是一个整齐规则库？
智能体是否必须追求统一、完美的逻辑系统？
```

关键理解：

> 理解一个故事，并不是单一能力，而可能需要语言理解、因果推理、情绪感受、社会经验、记忆联想等许多能力协同工作。

更准确的表达：

> 早期符号主义系统可以表现出某些局部智能，但很难通向开放世界里的通用智能。

### 14.2 作为协作体的智能

在心智社会里，`agent` 不是现代意义上完整的 Agent，而是一个非常简单、专门化的心智过程。

例如：

```text
LINE-FINDER：只负责识别线条
GRASP：只负责抓握
REACH：只负责伸手
SEE-SHAPE：只负责看形状
```

单个小 agent 不理解完整任务。`GRASP` 不知道什么是“搭塔”，它只知道抓。

多个小 agent 组合起来，可以形成更大的功能单元，也就是 `Agency`，机构。

例如一个 `BUILD` 机构可能包含：

```text
SEE
FIND
GET
PUT
```

核心概念是涌现 `Emergence`：

> 复杂智能行为不是某一个中心模块完整规划出来的，而是很多简单部分通过局部协作，自发形成的整体能力。

搭积木塔例子：

```text
BUILD-TOWER 激活 BUILDER
BUILDER 激活 ADD-BLOCK
ADD-BLOCK 依次激活 FIND-BLOCK、GET-BLOCK、PUT-ON-TOP
GET-BLOCK 又激活 SEE-SHAPE、REACH、GRASP
```

从外部看，系统像是“知道怎么搭塔”；但内部其实是很多小机制协作产生了整体行为。

### 14.3 对多智能体系统的理论启发

心智社会为后来的分布式人工智能 DAI 和多智能体系统 MAS 提供了概念基础。

它把研究焦点从：

```text
如何构建一个全能的单一智能体？
```

转向：

```text
如何设计一个高效协作的智能体群体？
```

三个重要启发：

| 启发 | 含义 |
| --- | --- |
| 去中心化控制 | 没有一个绝对中央控制器包办所有决策，多个智能体通过局部协作完成任务 |
| 涌现式计算 | 复杂解法可以从简单的局部交互规则中产生 |
| 智能体的社会性 | 智能体之间需要通信语言、交互协议、协商策略、信任模型和组织结构 |

今天纠正的理解：

> 去中心化控制不一定要求每个 agent 都在单独的计算机上。它可以是多台机器、多机器人，也可以是同一系统里的多个自治软件实体。重点是没有一个绝对中央控制器包办所有决策。

## 15. 2.4 学习范式的演进与现代智能体

心智社会指出了分布式协作的方向，但具体实现路径还不清晰。与此同时，符号主义的局限说明：仅靠人手写规则，很难构建真正鲁棒的智能系统。

于是出现一个关键问题：

> 如果智能无法被完全设计出来，那它能不能被学习出来？

学习范式的核心目标变了：

```text
不是手动编码知识
而是让系统从数据、经验、交互中自动获得知识和能力
```

## 16. 2.4.1 从符号到联结

联结主义 Connectionism 在 20 世纪 80 年代重新兴起，它和符号主义的思路正好相反。

符号主义是自上而下 `Top-down`：

```text
人先写好符号、规则、逻辑
-> 机器按规则推理
```

联结主义是自下而上 `Bottom-up`：

```text
先构造大量简单处理单元
-> 让它们通过连接形成网络
-> 通过数据训练调整连接权重
-> 整体网络学会识别模式
```

三个核心思想：

| 思想 | 含义 |
| --- | --- |
| 知识的分布式表示 | 知识不是明确规则，而是分散在大量神经元连接权重和激活模式中 |
| 简单的处理单元 | 单个人工神经元只做简单数值计算 |
| 通过学习调整权重 | 系统通过训练算法自动调整连接权重，让输出逐渐接近期望目标 |

今天专门纠正了一个点：

> 单个神经元通常不是稳定地负责一个清晰语义概念。更准确是：单个神经元做简单数值计算，很多神经元共同形成分布式表示，网络整体通过权重组合表达模式、关系和语义。

符号主义与联结主义的区别：

```text
符号主义：人类可读的规则，比如 IF-THEN。
联结主义：高维向量 + 权重矩阵 + 激活模式。
```

联结主义特别擅长解决高维、模糊、模式丰富的问题，比如图像识别、语音识别、文本语义理解。这些问题很难靠人手写完所有规则。

## 17. 2.4.2 基于强化学习的智能体

联结主义主要解决很多感知问题：

```text
这张图里有什么？
这段语音说了什么？
这段文本大概表达什么？
```

但智能体还要解决更核心的决策问题：

```text
在当前情况下，我应该做什么？
```

强化学习 Reinforcement Learning, RL 专注于序贯决策问题。

序贯决策指的是：

> 现在的动作会影响后面的状态，后面的状态又会影响下一步选择，不是一次判断就结束。

典型例子：

```text
训练一个游戏角色通关：
当前状态 = 角色位置、血量、敌人位置
行动 = 向左、向右、跳跃、攻击
奖励 = 吃到金币 +1，受伤 -1，通关 +100
目标 = 最大化整局游戏总奖励，而不是只追求眼前一步收益
```

强化学习的五个核心要素：

| 要素 | 含义 |
| --- | --- |
| Agent | 学习者和决策者 |
| Environment | 智能体交互的外部环境 |
| State | 当前环境状态，是决策依据 |
| Action | 智能体可以采取的操作 |
| Reward | 环境对行动结果给出的奖励或惩罚信号 |

强化学习循环：

```text
观察当前状态 S_t
-> 根据策略 Policy 选择行动 A_t
-> 环境转移到新状态 S_{t+1}
-> 环境给出奖励 R_{t+1}
-> 智能体根据新状态和奖励更新策略
-> 下一轮继续
```

关键理解：

> 强化学习追求的是长期累积奖励，而不是单步即时奖励。比如游戏里暂时绕路或避开金币，可能是为了未来通关获得更大奖励。

## 18. 2.4.3 基于大规模数据的预训练

强化学习让智能体能从交互中学习决策策略，但也有局限：很多 RL 智能体一开始缺少先验知识，需要从零开始理解任务，因此需要大量试错和针对特定任务的交互数据。

这引出一个问题：

> 能不能让智能体在开始具体任务前，就先具备广泛的语言知识、世界知识和常识背景？

这个问题在 NLP 领域通过预训练 Pre-training 得到突破。

### 18.1 预训练与微调

预训练-微调范式分两步。

第一步：预训练 `Pre-training`

```text
在海量通用文本上训练大模型
-> 用自监督学习从数据本身构造训练目标
-> 学习语言规律、事实知识、常识关联和上下文逻辑
```

典型目标：

```text
预测下一个词
```

第二步：微调 `Fine-tuning`

```text
在特定任务的小规模数据上继续训练
-> 让模型适应具体用途
```

今天重点区分了监督学习和自监督学习：

```text
监督学习：人额外给标签，比如“正面/负面”。
自监督学习：不需要人工标签，而是从数据本身自动构造训练目标，比如预测下一个词。
```

关键理解：

> 自监督预训练不是没有答案，而是答案藏在数据本身里，不需要人额外标注。

### 18.2 预训练如何缓解知识获取瓶颈

符号主义获取知识的方式：

```text
访谈专家
-> 提炼知识
-> 写成 IF-THEN 规则
-> 放进知识库
```

预训练换了一种方式：

```text
从海量文本中自动学习语言和世界知识
-> 把知识压缩进模型参数
```

所以预训练缓解的是“知识必须靠人工一条条编码”的问题。

但它不是完美解决：

- 互联网文本可能包含偏见；
- 知识可能错误；
- 信息可能过时；
- 模型可能幻觉。

## 19. 大型语言模型的诞生与涌现能力

原文指出，当模型在数万亿级文本上进行预训练后，神经网络权重中实际上形成了一个关于世界知识的、高度压缩的隐式模型。

### 19.1 隐式世界模型

隐式世界模型的意思是：

> 知识不是以人类可读的明文规则存放，而是以参数、权重、向量关系和激活模式的形式压缩在模型内部。

符号主义的知识是显式的：

```text
IF 发烧 AND 咳嗽 THEN 可能呼吸道感染
```

LLM 的知识不是这样存的。它没有一条可直接打开查看的规则写着：

```text
France capital = Paris
```

但模型在生成时可以表现出它知道：

```text
The capital of France is Paris.
```

关键区别：

```text
符号主义规则库：知识明文写在规则里，可读、可查、可追踪。
LLM 隐式模型：知识压缩在参数和向量关系里，不易直接解释，但能在生成中表现出来。
```

### 19.2 上下文学习

上下文学习 In-context Learning 指的是：

> 不修改模型权重，只在输入上下文里给出任务说明或少量示例，模型就能临时理解并完成新任务。

例子：

```text
苹果 -> apple
香蕉 -> banana
猫 -> cat
狗 -> ?
```

模型能从上下文看出这是中英翻译任务，然后回答：

```text
dog
```

如果给几个例子，叫 `few-shot`；如果不给例子，只直接提出任务，叫 `zero-shot`。

它的重要性：

> LLM 不需要为每个新任务都重新训练，可以通过上下文临时适应任务。

### 19.3 思维链

思维链 Chain-of-Thought 指的是：

> 让模型在回答复杂问题前，先展开中间推理步骤。

它能帮助模型做复杂推理的原因是：复杂问题如果直接输出最终答案，很容易跳步；把中间步骤展开后，模型更容易沿着问题结构推，也更方便人检查哪里出错。

今天纠正的理解：

> 思维链不一定是“减少随机性”，更准确是减少跳步和中间推理缺失。模型仍然可能出错，但推理过程更可检查。

本节关键结论：

> LLM 不再只是语言模型，而开始成为兼具海量知识库和通用推理引擎角色的组件。但它仍然只是现代 Agent 的核心组件之一，不等于完整 Agent。

## 20. 2.4.4 基于大语言模型的智能体

随着 LLM 的发展，以 LLM 为核心的智能体成为新范式。它不仅能理解和生成人类语言，更重要的是可以通过与环境交互，自主地感知、规划、决策和执行任务。

现代 LLM Agent 的工作流可以接回第一章的 Agent Loop：

```text
感知 -> 思考 -> 行动 -> 观察 -> 再感知 -> 再思考 -> 再行动
```

### 20.1 感知 Perception

感知阶段从外部环境接收原始输入，形成观察 Observation。

观察可以是：

```text
用户指令
API 返回结果
网页内容
工具执行结果
环境状态变化
```

### 20.2 思考 Thought

思考是智能体的认知核心，通常由规划模块、LLM 和记忆模块协同完成。

它会做几件事：

- 理解当前观察；
- 结合历史记忆；
- 把宏观目标拆成具体步骤；
- 进行推理和决策；
- 决定下一步要调用什么工具或执行什么行动；
- 必要时进行反思和自我批判。

### 20.3 行动 Action

行动阶段由执行模块负责。LLM 生成工具调用指令后，执行模块会解析指令、校验工具和参数，并真正调用外部工具。

例如 LLM 生成：

```text
Action: search_web(query="Datawhale Hello-Agents")
```

执行模块需要做：

```text
解析 tool_name = search_web
校验工具是否允许
校验参数是否合法
真正调用搜索工具
拿到结果
包装成 Observation 返回给 LLM
```

关键理解：

> LLM 生成的是文本形式的决策或工具调用意图，执行模块负责把它变成真实可执行动作，并承担权限、格式和参数控制。

### 20.4 观察 Observation 与循环

工具执行后会返回结果，这个结果构成新的观察，再进入下一轮感知、思考和行动。

所以现代 LLM Agent 不是一次性回答，而是持续迭代的闭环系统。

### 20.5 为什么 LLM 不等于完整 Agent

LLM 是现代 Agent 的认知核心之一，但它单独不等于完整 Agent，因为它自己不能可靠完成：

```text
感知环境
调用真实工具
改变外部状态
接收执行结果
根据结果继续循环
```

完整 Agent 还需要：

- 感知模块；
- 记忆模块；
- 规划模块；
- 执行模块；
- 工具箱；
- 环境反馈；
- 停止条件和失败兜底。

记忆模块的作用：

> 让 Agent 能保持长期任务状态、用户偏好和历史上下文。没有记忆，Agent 会像每一轮都重新认识用户，难以完成连续任务。

## 21. 2.4.5 智能体发展关键节点概览

原文最后对智能体发展史进行总览。智能体的发展不是一条直线，而是几大思想流派长期交织、竞争和融合的过程。

主要有三大思潮。

### 21.1 符号主义

核心观点：

> 智能来自符号操作、规则和逻辑推理。

代表成果：

- 专家系统；
- SHRDLU；
- 深蓝 Deep Blue。

给现代 Agent 留下的东西：

- 规则；
- 规划；
- 逻辑推理；
- 可解释流程；
- 工具调用中的形式化接口；
- 失败兜底中的硬约束。

### 21.2 联结主义

核心观点：

> 智能可以从大量简单单元连接成网络，并通过数据学习形成能力。

它不是多智能体协作，而是神经网络路线：

```text
大量简单神经元
-> 通过连接权重组成网络
-> 从数据中学习
-> 用高维向量、权重和激活模式表示知识
```

代表成果：

- 神经网络；
- CNN；
- Transformer；
- 大语言模型。

给现代 Agent 留下的东西：

- 向量表示；
- 神经网络；
- 语言模型；
- 模式识别；
- 大规模预训练；
- 隐式世界知识。

### 21.3 行为主义

核心观点：

> 智能体通过与环境互动、试错、奖励反馈和策略优化来学习行为。

它在现代 AI 中主要对应强化学习。

给现代 Agent 留下的东西：

- 环境交互；
- 奖励反馈；
- 策略学习；
- 长期累积奖励；
- 序贯决策；
- 通过行动结果调整后续行为的思想。

今天纠正的理解：

> 强化学习不是保证“得到全局最优解”，而是通过环境互动、奖励反馈和大量试错，优化长期累积奖励，学到更好的策略。

### 21.4 联结主义和心智社会的区别

今天重点纠正了一个混淆：

```text
联结主义：神经网络从数据中学习表示和规律，知识分布在权重、向量和激活模式里。
心智社会 / 多智能体：多个相对独立的智能体或模块协作、通信、分工，产生整体能力。
```

可以这样记：

```text
联结主义是一条神经网络和数据学习路线。
心智社会/多智能体是一种分布式协作思想。
```

### 21.5 现代 Agent 是多种思想的融合

现代 LLM Agent 不是单纯的“深度学习胜利”。

更准确地说：

```text
LLM / 神经网络提供语言、知识、推理能力
+ 符号系统提供规则、工具接口、结构化流程和兜底约束
+ 强化学习/行为主义提供反馈、试错和长期决策思想
+ 心智社会/多智能体提供协作和分工思想
```

所以现代 Agent 是多种历史思想在 LLM 时代的重新组合。

## 22. 2.5 本章小结

第二章的主线不是背历史人物或项目名称，而是理解现代智能体为什么会长成今天这样。

本章讲了四条线：

### 22.1 符号主义的探索与局限

早期 AI 试图用：

```text
知识 + 推理
```

来模拟智能。

贡献：

- 把智能从哲学问题变成可工程化的问题；
- 在专家系统、SHRDLU、ELIZA 等系统中展示了局部智能；
- 留下了规则、推理、规划、形式化接口等重要思想。

局限：

- 知识获取瓶颈；
- 常识问题；
- 框架问题；
- 系统脆弱性；
- 真实世界规则写不完。

### 22.2 分布式智能思想的萌芽

明斯基的心智社会提出：

```text
智能不是单一完美核心，
而是大量简单部分协作涌现。
```

这为后来的多智能体系统提供了理论启发。

### 22.3 学习范式的演进

智能获取能力的方式发生变化：

```text
不是手动编码知识，
而是从数据、经验、交互中学习。
```

包括：

- 联结主义：从数据中学习模式；
- 强化学习：从环境互动和奖励反馈中学习决策；
- 大规模预训练：从海量文本中学习语言、知识、常识和上下文逻辑。

今天纠正的理解：

> LLM 预训练不是“小规划”，而是在模型做具体任务前，先从海量文本中获得先验知识和通用语言能力，让模型不必每个任务都从零开始。

### 22.4 现代智能体的诞生

现代 LLM Agent 把历史上的多种思想融合起来：

```text
LLM：语言、知识、推理
Memory：历史上下文和长期信息
Planning：目标拆解和步骤规划
Tool Use：调用外部工具影响环境
Observation：根据结果继续调整
```

关键总结：

> 智能体的发展不是简单技术升级，而是一场围绕“如何定义智能、如何获取知识、如何做决策”的思想变革。

## 23. 我的学习轨迹：这章真正变成我的理解的地方

这章一开始我以为自己只是在学一段 AI 发展史，但真正有价值的地方，是把“为什么现代 Agent 需要这些组件”一步步问清楚了。

### 23.1 第一个卡点：为什么学习范式之后还需要 LLM Agent

我一开始对“问题驱动的演进”有疑问：既然学习范式已经能让 AI 从数据里学习，那到底是什么问题继续推动了 LLM 驱动 Agent 的出现？

后来这条线被拆清楚了：

```text
符号主义解决：智能如何被规则和逻辑工程化
但问题是：规则写不完，开放世界太复杂

学习范式解决：能力可以从数据和经验中学出来
但问题是：很多系统仍然偏专用，缺少通用语言接口和广泛世界知识

LLM 预训练解决：先从海量文本中获得语言、知识、常识和上下文推理能力
但 LLM 本身还不等于 Agent
```

这一段最后形成的理解是：

> LLM 更像现代 Agent 的“大脑候选”，它提供语言理解、知识压缩、上下文推理和任务泛化能力；当它接上记忆、规划、工具、环境反馈和停止条件之后，才成为 LLM 驱动的 Agent。

这个纠偏很重要，因为我曾经容易把“LLM 会用工具”理解成 LLM 自带的能力。更准确地说，工具使用是 Agent 架构通过提示词、函数调用、工具接口和执行循环接上的能力。

### 23.2 第二个卡点：符号主义不是“喂很多数据”

学符号主义时，我一开始会把“知识库很丰富”说成“喂了很多数据”。这个说法后来被纠正了。

符号主义的能力不是从大量训练数据里自动学出来的，而是来自：

```text
人类专家手工整理知识
-> 工程师编码成规则
-> 推理机按规则匹配和推导
```

所以专家系统的“聪明”来自人工编码的知识库和推理规则，而学习范式的“聪明”来自数据、经验和学习算法自动调整出来的模型参数或策略。

这个区分让我重新理解了 MYCIN、SHRDLU、ELIZA 的共同局限：它们不是“不够努力”，而是底层方法决定了它们很难覆盖开放世界。

### 23.3 第三个卡点：像理解，不等于真理解

ELIZA 这一段对我最有产品启发。它通过关键词匹配、分解规则、重组规则和代词转换，让用户产生“它懂我”的感觉。

但它本质上没有语义理解，也没有上下文记忆。

这让我留下了一个产品判断：

> 用户体验到“被理解”，不代表系统真的理解了用户目标。尤其在心理、医疗、法律、金融等高风险场景里，产品不能把“像理解”的交互效果包装成可靠智能。

这比单纯记住 ELIZA 的实现方式更重要。它提醒我：AI 产品设计里，要区分情绪上的拟人化体验和系统真实可验证的能力边界。

### 23.4 第四个卡点：心智社会、联结主义、多智能体不要混在一起

我曾经把“很多简单单元连接起来”和“很多智能体协作起来”混得有点近。

最后整理成了这个区别：

```text
联结主义：
神经网络从数据中学习表示和规律，知识分布在权重、向量和激活模式里。

心智社会 / 多智能体：
多个相对独立的模块或智能体协作、通信、分工，整体产生更复杂的能力。
```

我当时给自己的记法是：

> 联结主义是一条神经网络和数据学习路线；心智社会/多智能体是一种分布式协作思想。

现代 Agent 会同时借用两者：LLM 本身主要来自联结主义，而多 Agent 协作、任务分工、角色拆分，又借用了心智社会和多智能体思想。

### 23.5 最后一次综合纠偏：强化学习和预训练

最后综合题里，我把强化学习说成了“倒推出全局最优解”，也把 LLM 预训练说成了“小规划”。这两个点后来都被修正了。

强化学习更准确是：

```text
智能体与环境互动
-> 试错
-> 获得奖励或惩罚
-> 更新策略
-> 追求更高的长期累积奖励
```

它不保证全局最优，现实里也可能受奖励设计、探索不足和局部最优影响。

LLM 预训练更准确是：

```text
模型在具体任务之前
先从海量文本中学习语言规律、世界知识、常识关联和上下文逻辑
让后续任务不必从零开始
```

这次纠偏把第二章收束成了一个更清楚的判断：

> 现代 Agent 不是某一种技术路线的胜利，而是多种历史思想的重新组合。符号主义留下规则、接口和可解释流程；联结主义提供 LLM 的表示和生成能力；行为主义/强化学习留下环境反馈和长期决策思想；心智社会和多智能体思想留下分工协作的组织方式。

## 24. 第二章最终理解

如果用一句话总结第二章：

> 现代智能体长成今天这样，是因为单一范式都解决不了开放世界里的复杂任务，所以它把 LLM 的通用语言和知识能力、符号系统的结构化规则与工具接口、强化学习的环境反馈思想、多智能体的协作思想组合到了一起。
