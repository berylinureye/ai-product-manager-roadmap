# 2026-06-16 AI PM 岗位定位笔记

## 1. 我主攻的方向

短期两个月内，我主攻：

- AI 应用型产品经理
- AI 软件型产品经理

这两个方向更看重场景需求挖掘、产品设计、技术选型、跨团队推进、数据验证和 AI 功能落地，和我当前的经历更匹配。

我会把 `Agent / RAG 应用能力` 作为差异化能力来补，但不把自己定位成底层算法或模型训练方向。

## 2. 为什么不是基座模型产品

基座模型产品更靠近底层算法、训练、推理、算力、模型评测和平台能力，对算法理解和工程深度要求很高。

我当前需要补充的内容太多，如果把两个月冲刺主线放在基座模型产品上，投入产出比不高。

当前更合理的选择是：

- 了解基座模型能力边界。
- 会比较不同模型和供应商。
- 会把模型能力转化成具体产品方案。
- 会设计评测集、bad case 归因和 fallback。

## 3. 为什么不是软硬结合型产品

软硬结合型产品是朝阳方向，尤其适合 AI 硬件、具身智能、教育硬件、陪伴机器人等场景。

但这个方向会额外要求：

- 供应链理解
- BOM 规划
- 硬件工业设计
- 工程设计
- 工厂协作
- 开模试机
- 可靠性测试
- 小规模生产和量产流程

我目前对硬件知道得太少，所以短期不把它作为主攻方向。中长期可以有意识补充供应链和硬件产品知识，作为未来向 AI 硬件或软硬结合产品靠拢的储备能力。

## 5. 我最需要补的能力

### 1. 最新 Agent 相关知识

需要补：

- Agent 的基本结构：目标、工具、记忆、规划、执行、反思、兜底。
- Agent 的常见范式：ReAct、Plan-and-Solve、Reflection。
- Agent 的失败模式：工具调用失败、上下文污染、循环执行、目标漂移、输出不可控。
- Agent 的产品指标：完成率、人工接管率、重试率、满意度、时延、成本。

学习材料：

- Coze / Dify 实操
- Datawhale Hello-Agents

### 2. 1-2 个新的作品集

需要补：

- 一个 Agent/RAG demo。
- 一个 AI 产品 PRD。

优先方向：

- GitHub 周榜分析 Agent
- AI 提升跨境下单转化 PRD
- A-Level Plus 拍照批改优化

### 3. 最新模型认知

需要补：

- 最新主流模型的能力差异。
- 不同模型在写作、推理、长上下文、代码、多模态、工具调用上的差异。
- 模型选型如何影响成本、时延、稳定性和产品体验。

输出方式：

- 做一篇模型体验对比笔记。
- 用同一个任务测试 3-5 个模型。
- 把结果转成“产品选型建议”。

### 4. 最新 AI 产品观察

需要补：

- 每周看 GitHub 周榜。
- 每周体验 1-3 个新 AI 产品。
- 记录产品定位、用户场景、增长原因、交互亮点和可复用点。

目标不是追热点，而是把优秀产品的优势融入自己的作品里。

> 我能理解 AI 能力边界，能选择合适模型和工具，能把 Agent/RAG 落进真实场景，并用数据和评测证明产品有效。
