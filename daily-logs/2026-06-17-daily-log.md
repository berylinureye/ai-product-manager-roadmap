## 今天学了什么

- Agent 不是“会聊天的模型”，而是围绕目标与环境交互的系统。
- Agent 的核心闭环是：感知环境、思考下一步、执行动作、观察结果、再调整。
- 传统智能体从简单反射、基于模型、基于目标、基于效用到学习型逐步演进。
- LLM Agent 的新范式是：用大语言模型理解自然语言目标、规划步骤、调用工具，并根据反馈动态修正。
- PEAS 可以用来定义 Agent 产品：性能度量、环境、执行器、传感器。
- Agent Loop 和 TAO 协议解释了 Agent 如何通过 Thought、Action、Observation 进行工具调用和状态更新。

## 今天纠正的误区

- 多模态模型不等于 Agent，多模态只是传感器能力。
- 网页总结器不一定是 Agent，只有形成目标、行动、观察和下一步决策闭环才更像 Agent。
- README 总结属于亚符号主义能力，因为它需要语义理解。
- Parser 不是思考模块，而是解析 Action 并交给工具执行的外部程序。
- Finish 不只是输出格式，而是 Agent 停止循环的信号。
