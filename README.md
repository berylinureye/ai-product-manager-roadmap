# AI Product Manager Learning Roadmap

这个仓库用于记录我转向 AI 产品经理过程中的系统学习、章节笔记、产品思维训练和核心概念笔记。现阶段先专注学习，不把作品集、GitHub 排行榜分析或独立项目混在这个 roadmap 里。

## 当前学习目标

- 建立 AI 产品经理需要的基础语言系统：LLM、Prompt、Agent、RAG、MCP、评测集、bad case、fallback、成本与效果取舍。
- 用 Datawhale、B 站课程和低代码工具补齐 Agent/RAG 的原理与产品判断。
- 通过章节笔记和产品 case 训练，沉淀自己的理解、错误纠偏和可复用表达。

## 我对产品思维的理解

我现在对产品思维的理解是：产品经理不是把一个需求直接做成一个功能，而是先判断这个问题是否真实、是否重要、是否和公司当前目标一致，以及它是否值得投入资源。

面对一个需求，我需要先问清楚：

- 用户是谁，在什么场景下遇到了什么问题；
- 这个问题是否高频、高痛，现有替代方案为什么不够好；
- 它能服务公司的哪个目标，例如留存、转化、收入、效率或战略方向；
- 第一阶段能不能用更轻的 MVP 验证，而不是一开始做大而全的方案；
- 核心指标是否足够贴近功能本身，风险指标是否能约束错误体验、隐私和信任问题。

所以我想训练的不是“想出更多功能”，而是建立一套判断顺序：先证明问题值得做，再决定做什么；先用近因指标验证价值，再看长期业务结果。

## 记录规则

- 系统教程按资料来源单独建文件夹，例如 `datawhale-hello-agents/`。
- 每次学习结束后更新对应章节笔记，并在笔记里记录日期、理解、纠偏和下次继续位置。
- 独立项目、GitHub 排行榜分析、作品集材料暂时不放在这个 roadmap 里。
- 每篇笔记尽量记录：学了什么、我怎么理解、哪里答错或混淆、下一步要补什么。
