# 8 Week AI Product Manager Learning Plan

起始日期建议：2026-06-15

节奏：每天约 3 小时，每周 6 天学习 + 1 天复盘补漏。目标不是“泛泛学 AI”，而是在 8 周后拥有可投递、可面试、可展示的 AI 产品经理证据链。

## 总路线

第 1-2 周：补齐 AI PM 语言系统，建立模型边界、Prompt、产品拆解、PRD 基本盘。

第 3-4 周：做出 Agent/RAG demo，把“懂 AI”变成“能跑通”。先用 Coze/Dify/RAGFlow 建立手感，再用 Datawhale 的 `hello-agents` 和 `all-in-rag` 补系统原理。

第 5-6 周：把你的金融策略、教育 AI、跨境转化方向包装成作品集。

第 7-8 周：集中求职化：简历、面试话术、岗位投递、GitHub 主页和每周记录。

## Week 1: AI 产品认知与模型边界

目标：能说清 AI 产品经理到底在判断什么：能力边界、成本、效果、用户价值、风险。

Day 1:

- 学习：大模型基础概念，理解 LLM、上下文窗口、幻觉、token、推理、工具调用。
- 动手：体验 3 个模型或 AI 产品，例如 ChatGPT/Claude/DeepSeek/豆包/Kimi，分别做同一个任务。
- 输出：写 `Week 1 - 模型能力对比.md`，记录“谁适合写作、谁适合推理、谁适合长文档、谁更容易胡说”。

Day 2:

- 学习：AI 产品经理 3 类岗位：应用型、Agent/工作流、AIGC/多模态。
- 动手：找 5 个 AI PM JD，提取高频要求。
- 输出：做一张“JD 能力词表”：LLM、Agent、RAG、SQL、PRD、评测、数据分析。

Day 3:

- 学习：Prompt 工程，重点是角色、上下文、任务、格式、示例、约束、评估。
- 动手：为 A-Level Plus 写 3 版批改 prompt，并定义比较标准。
- 输出：记录 prompt 版本、效果差异、失败样例。

Day 4:

- 学习：AI 功能该不该做，比较规则方案、传统 ML、大模型方案。
- 动手：用美团月付或跨境下单转化场景做一次方案对比。
- 输出：一页方案判断表：用户问题、传统方案、AI 方案、成本、风险、指标。

Day 5:

- 学习：AI 产品评测集：准确率、可用率、满意度、bad case、人工抽检。
- 动手：为 A-Level Plus 设计 20 条小型评测集。
- 输出：评测集表格字段：题目类型、标准答案、模型答案、错误类型、修复策略。

Day 6:

- 学习：结构化表达和金字塔原理。
- 动手：把本周内容整理成一篇 GitHub 周记录。
- 输出：第一篇周报，标题建议：`Week 1: 从金融策略产品到 AI PM，我补齐了哪些判断语言`。

Day 7:

- 复盘：整理不会的概念，补 1 小时视频/文档。
- 维护：更新 GitHub README 或学习仓库目录。

## Week 2: 产品基本功与数据基本盘

目标：把你已有的产品和数据能力翻译成 AI PM 能读懂的材料。

Day 1:

- 学习：PRD 结构：背景、目标、用户故事、流程、功能、指标、边界、排期。
- 动手：选择一个作品方向：`AI 提升跨境下单转化` 或 `A-Level Plus 拍照批改优化`。
- 输出：PRD v0.1 大纲。

Day 2:

- 学习：竞品拆解方法：用户、场景、路径、功能、指标、商业模式。
- 动手：拆 2 个教育 AI 或 Agent 产品。
- 输出：竞品拆解表。

Day 3:

- 学习：SQL 漏斗、留存、转化、分层。
- 动手：用模拟数据设计跨境下单转化漏斗，至少包含访问、加购、结算、支付成功。
- 输出：3 条 SQL 示例 + 指标解释。

Day 4:

- 学习：埋点设计。
- 动手：为 A-Level Plus 设计埋点：拍题上传、OCR 成功、批改完成、追问、满意/不满意。
- 输出：埋点表 + 北极星指标。

Day 5:

- 学习：AI 产品的成本结构：模型调用、响应时延、人工审核、失败兜底。
- 动手：估算 A-Level Plus 单次批改成本和 1000 次/日成本。
- 输出：成本测算表。

Day 6:

- 学习：GitHub 周榜观察方法。
- 动手：从 OpenGithubs 周榜选 3 个 AI/Agent/PM 相关项目体验或阅读 README。
- 输出：第一篇 `GitHub Rank Review`。

Day 7:

- 复盘：把 PRD v0.1 改成 v0.2，补充指标和风险。

## Week 3: Agent 与工作流编排

目标：亲手跑通一个可演示 Agent，简历可以写“设计并落地”。

阶梯材料：

- 入门手感：Coze/扣子教程，先跑通一个低代码 Agent。
- 系统原理：Datawhale `hello-agents`，先读第 1 章“初识智能体”、第 4 章“智能体经典范式构建”、第 5 章“基于低代码平台的智能体搭建”。

Day 1:

- 学习：Agent 基本结构：目标、记忆、工具、规划、执行、反思、兜底。
- 动手：画出 A-Level Plus 多 Agent 批改流程图。
- 输出：架构图 + 每个 Agent 的职责。

Day 2:

- 学习：Coze/扣子或 Dify 的工作流节点。
- 动手：搭一个“AI 产品周榜分析 Agent”：输入 GitHub 项目链接，输出产品价值、目标用户、PM 可学点。
- 输出：Agent 截图、prompt、工具列表。

Day 3:

- 学习：工具调用/function calling。
- 动手：给 Agent 增加搜索、网页读取或表格输出能力。
- 输出：记录成功路径和失败路径。

Day 4:

- 学习：fallback 设计。
- 动手：设计 5 种失败兜底：搜索失败、链接打不开、内容太长、结论不可信、输出格式错。
- 输出：fallback 表。

Day 5:

- 学习：Agent 体验指标：完成率、重试率、人工接管率、满意度、平均时延。
- 动手：跑 10 个 GitHub 项目链接，统计 Agent 输出质量。
- 输出：小型评测表。

Day 6:

- 输出：录一段 30 秒 demo 讲解稿，写成文字版也可以。
- GitHub：提交 demo README。

Day 7:

- 复盘：修 prompt、补流程图；写一页 Agent 原理笔记，讲清 ReAct、Plan-and-Solve、Reflection 分别解决什么问题。

## Week 4: RAG 与知识库

目标：能讲清 RAG 全链路，并做一个知识库问答 demo。

阶梯材料：

- 入门手感：Dify/RAGFlow，先搭一个知识库问答 demo。
- 系统原理：Datawhale `all-in-rag`，先读第 1 章“解锁 RAG”、第 2 章“数据准备”、第 3 章“索引构建”、第 6 章“RAG 系统评估”。

Day 1:

- 学习：RAG 全链路：切块、向量化、召回、重排、生成、防幻觉。
- 输出：一张 RAG 架构图。

Day 2:

- 动手：用 Dify/RAGFlow/Coze 搭一个简历知识库或 A-Level 题库知识库。
- 输出：知识库配置记录。

Day 3:

- 学习：chunk 策略和召回失败。
- 动手：同一份材料用不同切块方式测试。
- 输出：bad case 记录。

Day 4:

- 学习：防幻觉策略：引用来源、拒答、置信度、人工审核。
- 动手：为知识库问答设置“不知道就拒答”的规则。
- 输出：拒答策略说明。

Day 5:

- 学习：RAG 在企业落地场景：客服、销售、法务、培训、投研。
- 动手：把美团策略周报或金融投研场景改造成一个 RAG 产品方案。
- 输出：一页产品方案。

Day 6:

- GitHub 周榜：继续选 3 个项目，重点看 Agent/RAG/MCP。
- 输出：第二篇周榜观察。

Day 7:

- 复盘：把 Agent + RAG demo 合并成作品集条目；写一页 RAG 评估笔记，覆盖检索命中、答案忠实度、引用准确性、拒答能力。

## Week 5: 作品集 1 - A-Level Plus AI 拍照批改

目标：把已有项目包装成面试作品，而不是只停留在项目描述。

本周前半段先做 Agent/RAG 进阶补强，后半段进入作品集包装：

- `hello-agents` 选读第 8 章“记忆与检索”、第 10 章“智能体通信协议”、第 12 章“智能体性能评估”。
- `all-in-rag` 选读第 4 章“检索优化”，重点看混合检索、查询构建、查询重构。
- 输出一篇 `Agent + RAG 为什么会失败，以及产品经理怎么兜底`，再把这些语言塞回 A-Level Plus 项目。

Day 1:

- 补强：阅读 Hello-Agents 记忆与检索章节，把 A-Level Plus 的“追问”和“错题知识库”映射到记忆/RAG。
- 输出：A-Level Plus 的记忆/检索设计说明。

Day 2:

- 补强：阅读 Hello-Agents 协议/评估章节，整理 Agent 失败模式。
- 输出：`Agent + RAG 为什么会失败，以及产品经理怎么兜底`。

Day 3:

- 梳理：用户是谁、痛点是什么、为什么需要 AI。
- 输出：项目背景和用户旅程。

Day 4:

- 梳理：多模型协作、5 个 Agent 投票、Python 规则校验器。
- 输出：技术流程图，重点讲产品取舍，不讲成纯工程项目。

Day 5:

- 梳理：评测集、bad case、错误归因、指标体系。
- 输出：至少 10 个 bad case + 北极星指标 + 过程指标。

Day 6:

- 梳理：商业化与增长。
- 输出：作品集文档 v1，包含定价假设、获客渠道、留存机制、转介绍机制。
- GitHub：整理 README。

Day 7:

- 复盘：用 STAR 法准备 3 分钟项目讲述。

## Week 6: 作品集 2 - AI 提升跨境下单转化

目标：把本地已有的跨境转化主题发展成 AI PM 作品，体现你的金融/增长/数据优势。

Day 1:

- 梳理：跨境下单链路，找出 3 个转化阻塞点：信息不信任、支付/物流不确定、客服响应慢。
- 输出：用户旅程和漏斗图。

Day 2:

- 方案：AI 导购/客服 Agent。
- 输出：功能清单：商品解释、尺码/参数答疑、物流税费解释、优惠推荐、支付失败辅助。

Day 3:

- 数据：设计漏斗指标和实验方案。
- 输出：A/B 实验设计：转化率、客单价、客服转人工率、满意度、退款率。

Day 4:

- 风险：幻觉、合规、价格错误、承诺过度。
- 输出：风险与兜底策略。

Day 5:

- Demo：用 Coze/Dify 做一个跨境导购 Agent 原型。
- 输出：prompt、工具、知识库、样例对话。

Day 6:

- GitHub 周榜：选 3 个与电商、Agent、AI 搜索、增长相关项目。
- 输出：第三篇周榜观察。

Day 7:

- 复盘：形成 PRD v1。

## Week 7: 求职材料与面试

目标：把学习成果转成可投递资产。

Day 1:

- 重写简历标题：`AI 产品经理 / 金融策略产品 / Agent & RAG 应用`。
- 输出：新版简历第一版。

Day 2:

- 简历项目改写：美团经历突出策略、指标、AI 提效；A-Level Plus 突出 AI 产品落地。
- 输出：2 个版本：AI 应用 PM、金融 AI PM。

Day 3:

- 准备面试问答：为什么转 AI PM、你的 AI 项目怎么评估、Agent 失败怎么办。
- 输出：10 个高频问答。

Day 4:

- 准备技术解释：LLM、RAG、Agent、MCP、Prompt、评测集，用产品语言讲。
- 输出：一页技术解释小抄。

Day 5:

- 岗位研究：找 20 个岗位，按匹配度分 A/B/C。
- 输出：投递清单。

Day 6:

- GitHub：整理主页仓库 README，让别人 30 秒知道你在转 AI PM。
- 输出：主页说明草稿。

Day 7:

- 复盘：模拟面试 30 分钟，记录卡壳点。

## Week 8: 冲刺与投递

目标：完成作品集闭环，开始真实投递。

Day 1:

- 完成作品集总文档：两个项目 + 周榜观察 + 学习路线。
- 输出：作品集目录。

Day 2:

- 优化 Agent/RAG demo，补截图和演示脚本。
- 输出：demo README 最终版。

Day 3:

- 优化简历，删掉不服务 AI PM 的弱信息。
- 输出：简历 v2。

Day 4:

- 投递 5 个 A 类岗位，记录 JD、投递时间、简历版本。
- 输出：投递表。

Day 5:

- 投递 10 个 B 类岗位，继续改关键词。
- 输出：投递表更新。

Day 6:

- 写第 8 周总结：两个月我补齐了什么、做出了什么、下一阶段怎么迭代。
- 输出：最终复盘。

Day 7:

- 面试准备日：项目讲述、技术解释、数据题、行为面试。

## 每周固定动作

- 每周至少 1 次 GitHub 周榜观察，选择 3-5 个 AI/Agent/PM 相关项目。
- 每周至少 1 次产品体验，记录：目标用户、核心场景、好用点、糟糕点、可借鉴点。
- 每周至少 1 个可展示产物：PRD、流程图、评测集、demo、SQL、竞品拆解、面试问答。
- 每周更新一次 GitHub，哪怕只是文档 commit。

## 推荐优先级

最高优先级：

1. A-Level Plus 作品集化。
2. Agent/RAG demo。
3. GitHub 周记录。
4. 新版 AI PM 简历。

次优先级：

1. LangChain 代码深挖。
2. MCP 原理扩展。
3. 多模态/AIGC 创作。

暂时不要分散精力：

- 不要一开始啃深度学习数学推导。
- 不要把 LangChain 当主线，除非目标岗位明确要求。
- 不要同时开太多 demo，先把 1-2 个讲透。
